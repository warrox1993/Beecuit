# BeeCuit — Phase 4.1 « Visual Polish » — Design

**Date :** 2026-05-25
**Statut :** Spec validée, plan à écrire
**Sprint parent :** Phase 4 « Améliorations visuelles » (3 sprints : 4.1 polish, 4.2 signature biscuit, 4.3 vidéo)
**Phase précédente :** Phase 2 B2B (Phase 3 3D abandonnée le 2026-05-25)

## Contexte & motivation

Après le pivot 2D, le site BeeCuit fonctionne mais reste visuellement statique : tout apparaît d'un bloc, pas de feedback animé sur les interactions, pas de hiérarchie de mouvement. L'objectif de ce premier sprint visuel est de **donner du souffle** à l'interface existante sans toucher au contenu : motions subtiles, feedback satisfaisant, polish header/footer, sans introduire de dépendances lourdes ni casser l'accessibilité.

## Objectifs

- Animations subtiles partout : reveal-on-scroll, hover cards, page transitions (P0).
- Feedback satisfaisant à chaque interaction clé : fly-to-cart + toast + badge bump (P0).
- Header sticky qui se réduit au scroll (P1).
- Nav underline animée, lazy image fade-in, scroll-progress sur pages longues, parallax Hero, dividers ornement, footer wave (P1).
- 100 % respect `prefers-reduced-motion`.
- Budget bundle : ≤ +50 KB First Load JS (cible totale `/` ≤ 250 KB).

**Hors scope :** signature biscuit (curseur miettes, cookie-bite scroll, etc. — sprint 4.2), contenu vidéo (sprint 4.3), refonte palette ou typographie.

## Stack & contraintes

**Nouvelles dépendances :**
- `framer-motion@^11` (~30 KB gzip) — orchestration des animations complexes.
- `sonner@^1.5` (~8 KB gzip) — toast notifications.

**Restent inchangées :** Next.js 15 App Router, Tailwind v4, shadcn/ui (button, dropdown, sheet), lucide-react, radix-ui primitives, next-intl.

**Aucune migration DB. Aucune nouvelle env var.**

## Décisions architecturales

| Question | Décision | Pourquoi |
|---|---|---|
| Lib animation | **Framer Motion** (vs GSAP / Auto-Animate) | déjà standard React, bonne intégration `prefers-reduced-motion`, AnimatePresence pour page transitions. |
| Lib toast | **Sonner** (vs react-hot-toast / radix-toast) | minimal, branding-friendly, accessibility-first, déjà compatible Next 15. |
| Page transitions | `AnimatePresence` dans `app/[locale]/layout.tsx` keyed sur pathname | natif Next App Router + Framer. Pas de routeur custom. |
| Fly-to-cart | Mini `motion.div` imperative créé au clic, animé fixed→target | pas de overlay React global, simple à mount/unmount. |
| Toast trigger | `toast.success()` de Sonner directement dans les server actions clients | pattern Sonner standard. |
| Hooks | Centralisés dans `hooks/` | conforme structure Next idiomatique. |
| Reduced motion | Framer `useReducedMotion()` hook + CSS `@media (prefers-reduced-motion)` pour CSS-only effets | source de vérité unique, hooks React-side. |
| Scope par fichier | 1 composant motion par concept | bornes claires, file size sain. |

### Spécifications visuelles précises

**Reveal-on-scroll (`MotionFadeUp`)** :
- Variants : `hidden: { opacity: 0, y: 20 }`, `visible: { opacity: 1, y: 0 }`.
- Durée : 0.5s ease-out.
- Trigger : `viewport: { once: true, amount: 0.2 }`.
- Reduced motion : skip → `visible` direct.

**Hover ProductCard / CoffretCard** :
- Translation : `-translate-y-1` (4 px).
- Shadow : `shadow-2xl` (Tailwind).
- Ring : `ring-2 ring-honey/30`.
- Image : `scale-[1.03]` sur `<img>` interne.
- Transitions : `transition-all duration-300 ease-out`.
- Reduced motion : skip translation + scale, garder shadow + ring.

**Fly-to-cart** :
- Source : DOM rect de la `<AddToCartButton>` cliquée.
- Cible : DOM rect de l'icône panier dans `<StickyHeader>` (via ref hook).
- Élément : `<img>` 32 × 32 px du produit (ou placeholder biscuit SVG si pas d'image).
- Trajectoire : arc parabolique 600 ms, easeOutCubic (formule réutilisable, ~15 lignes pures).
- Au complete : remove l'élément, déclencher `<CartBadge>` bump.
- Reduced motion : skip entièrement — juste toast.

**Page transition** :
- AnimatePresence `mode="wait"`.
- Children fade-out 0.15s puis fade-in 0.15s.
- Key = `usePathname()`.
- Reduced motion : skip → render direct.

**Sticky header** :
- Hauteur initiale : 80 px ; scrolled : 56 px.
- Background : `bg-cream` → `bg-cream/85 backdrop-blur-md`.
- Trigger : `window.scrollY > 50`.
- Transition : `transition-all duration-200 ease-out`.

**Toast (Sonner)** :
- Position : `top-right` desktop, `top-center` mobile.
- Theme custom : background `--color-cream`, border `--color-honey-dark`, text `--color-warm-brown`, icon honey.
- Durée par défaut : 3 s.

**Cart badge bump** :
- À chaque changement de count : `animate={{ scale: [1, 1.3, 1] }}`, 0.35 s.
- Reduced motion : skip.

**Nav underline** :
- CSS pur : `::after` pseudo-element width 0 → 100 %, `transition: width 200ms ease-out`.
- Couleur : honey-dark.
- Au hover du `<Link>`.

**Lazy image fade-in** :
- `<LazyImage>` initial `opacity-0`, après `onLoadingComplete` → `opacity-100`.
- Transition CSS `transition-opacity duration-300`.

**Scroll progress** :
- `<div className="fixed top-0 left-0 h-[2px] bg-honey z-50" style={{width: progress + '%'}} />`.
- Hook `useScrollProgress()` qui calcule `(scrollY / (docHeight - viewportH)) * 100`.
- Affiché seulement sur les pages avec `<ScrollProgress>` mounté.

**Hero parallax** :
- `useScroll()` + `useTransform()` Framer.
- Image translateY : `0 → -40px` sur la plage scroll [0, viewport.h].
- Reduced motion : skip.

**Section dividers (3 variants SVG inline)** :
- `sprig` : une branche florale stylisée 80 × 24 px, honey-dark stroke 1.5.
- `flourish` : ligne avec arabesque centrale 120 × 16 px.
- `dots` : trois losanges alignés 60 × 10 px.
- Tous centrés, `<div className="my-12 flex justify-center text-honey-dark">`.

**Footer wave** :
- SVG path scalloped (5-7 arcs) sur toute la largeur, fill = cream (couleur du body), placé au top du `<footer>` avec `margin-top: -32px`.
- Hauteur 32 px, responsive width 100 %.
- Effet : transition visuelle ondulée entre la dernière section et le footer.

## Critères d'acceptation (smoke local)

1. **Homepage** scroll → Hero apparaît au load, puis Featured, Story, Coffrets, Instagram, Newsletter fade-up en cascade au passage du viewport.
2. **ProductCard** hover → carte monte légèrement + ombre dorée + image zoom imperceptible.
3. **AddToCartButton** clic → mini biscuit s'envole de la carte vers l'icône panier (arc parabolique 600 ms), badge pulse, toast Sonner "Sablé ajouté au panier" top-right pendant 3 s.
4. **Header** au scroll > 50 px → shrink de 80 → 56 px, backdrop-blur visible, smooth transition.
5. **Nav** "Coffrets" hover → barre dorée slide-in 200 ms sous l'item.
6. **Page transition** clic sur "Notre histoire" → crossfade 0.3 s total au lieu de flash.
7. **Lazy image** : ouvrir DevTools throttling Slow 3G, les images fade-in au lieu de pop.
8. **Hero** scroll → image se translate doucement vers le haut.
9. **`/journal`** ouvrir un article long → filet doré progressif en haut visible.
10. **Section dividers** visibles entre StoryTeaser/Coffrets, Coffrets/Insta, Insta/Newsletter.
11. **Footer** : wave/scallop visible au top, fond cream continu.
12. **Reduced-motion** DevTools → animations désactivées, fonctionnel.
13. **Lighthouse perf** mobile ≥ 85.
14. **Bundle `/`** ≤ 250 KB First Load JS.
15. `pnpm tsc --noEmit && pnpm lint && pnpm vitest run tests/unit && pnpm build` tous verts.

## Risques & mitigation

| Risque | Mitigation |
|---|---|
| Page transitions + revalidation data Next | Tester sur routes server components (coffrets) ; si conflit, `mode="popLayout"` ou désactiver pour le routing data-heavy. |
| Toast positionning mobile vs sticky header | Configurer offset Sonner pour passer SOUS le header sticky (top: 80 px). |
| Bundle dépassement | Audit après chaque task ; envisager dynamic import pour Toaster si pas critique au first paint. |
| Animations cumulées qui font "lourd" | Coefficient global durée × 0.85 si retour utilisateur "trop animé". |
| `useReducedMotion` qui rate certains effets CSS pur | Backup via `@media (prefers-reduced-motion: reduce) { * { transition: none !important; } }` ciblé. |
